# Step 2 概念说明

Step 1 和 Step 2 做的事**一样**（三块 UI、坐标手写），只是代码组织方式变了。  
下面四个名字是 Step 2 才出现的，Step 1 里没有对应 import，但概念已存在。

## 对照 Step 1

Step 1（`step1_paint.py`）里有一段：

```python
rects = [
    (24, 24, 432, 48, "#dce8ff", "标题区"),
    (24, 88, 432, 120, "#e8f5e9", "内容区"),
    ...
]
for x, y, w, h, color, label in rects:
    p.drawRoundedRect(x, y, w, h, 6, 6)
```

Step 2 把「每一块」拆成独立对象，再挂到树上统一 `paint`。

---

## Rect — 矩形

```python
from miniui.geometry import Rect

Rect(x, y, width, height)
```

| 字段 | 含义 |
|------|------|
| `x`, `y` | 左上角在窗口里的坐标 |
| `width`, `height` | 宽和高 |

等价于 Step 1 的 `(24, 88, 432, 120)`，只是打包成一个对象，方便传给节点。

---

## Node — 树节点（基类）

```python
from miniui.node import Node
```

**UI 框架里，屏幕上的一切都是树里的结点。**

每个 `Node` 至少有：

| 属性/方法 | 作用 |
|-----------|------|
| `rect` | 这块 UI 占屏幕哪一块矩形 |
| `layout(rect)` | 父节点（或你自己）把 rect 交给它 |
| `paint(painter)` | 在 `rect` 里真正画出来 |
| `measure(...)` | Step 3 才用：汇报「我需要多大」 |

Step 2 **还没有** Column 自动排版，所以 `layout(Rect(...))` 只是**手动存坐标**，和 Step 1 手写数字是一回事。

---

## Text — 画文字的节点

```python
from miniui.widgets import Text

title = Text("你好", font_size=16, bold=True)
title.layout(Rect(32, 32, 400, 28))   # 告诉它：画在这个矩形里
# 真正绘制发生在 paintEvent → root.paint() → title.paint()
```

---

## Box — 画色块的节点

```python
from miniui.widgets import Box

body = Box(color="#e8f5e9", label="内容块")
body.layout(Rect(32, 80, 400, 100))
```

对应 Step 1 里的 `drawRoundedRect` + 背景色。

---

## 渲染树长什么样

```
ManualRoot                    ← 根，不画内容，只遍历孩子
  ├─ Text("Step 2 · …")       ← 叶子，有自己的 paint
  ├─ Box("内容块")
  └─ Text("提示")
```

**在哪里渲染？**

```text
QWidget.paintEvent
    └─ root.paint(painter)
           ├─ title.paint(painter)   ← 在 title.rect 里 drawText
           ├─ body.paint(painter)    ← 在 body.rect 里 drawRoundedRect
           └─ hint.paint(painter)
```

你写的 `Text(...)` / `Box(...)` **不会立刻出现在屏幕上**；  
窗口刷新时 `paintEvent` 走一遍树，才画出来。

## Step 2 → Step 3 差什么

| | Step 2 | Step 3 |
|--|--------|--------|
| rect 谁算 | 你手写 `Rect(32, 80, …)` | `Column` 根据 `spacing` 自动算 y |
| 改间距 | 改多处 y 坐标 | 只改 `spacing=12` |
